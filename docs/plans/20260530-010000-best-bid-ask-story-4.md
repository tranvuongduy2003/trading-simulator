## Executive summary

Deliver **Story 4** of US-05: when the **AAPL** order book is empty or one-sided, the top-of-book strip must show **explicit empty-state copy** (not misleading zeros or stale prices). Stories 1–3 already derive `null` touches and render spread/mid as **—**; this plan adds **liquidity-state–aware labels** (“No bids”, “No asks”, “No market — waiting for liquidity”), purges market snapshot cache on **logout**, and adds **integration tests** for bid-only and ask-only API snapshots. No new REST or SignalR contracts; simulated liquidity seeding stays out of scope (PRD §11.2 / engine).

**Definition of done:** All Story 4 acceptance criteria pass via automated API tests where applicable and a manual Aspire checklist; Stories 1–3 regression (live prices, real-time, spread/mid) unchanged.

## Goals and non-goals

**Goals**

- G1: No open bids → Best bid shows **“No bids”** (or **—** per spec); ask still shows live price when present (EC-02).
- G2: No open asks → Best ask shows **“No asks”**; bid still shows when present.
- G3: Empty book (both sides) → both sides show empty copy; strip-level hint **“No market — waiting for liquidity”**; spread/mid remain **—** (BR-03/BR-04 N/A).
- G4: API empty snapshot or `OrderBookUpdated` with empty `bids`/`asks` → UI reflects empty state until next non-empty update (no stale opposite-side prices).
- G5: Logout → order book query removed from TanStack Query cache so re-login does not flash prior session prices.
- G6: Manual Aspire checklist signed off.

**Non-goals** (this plan will not do)

- NG1: Simulated liquidity provider or engine book seeding.
- NG2: Multi-symbol empty states (MVP is **AAPL** only).
- NG3: Spread/mid formulas (Story 3, #60).
- NG4: New backend fields for “book state” on API/hub payloads.
- NG5: Frontend unit-test harness (Vitest) — derivation stays in pure TS for future tests; MVP uses manual UI per `core.mdc`.

## Implementation details (for /build)

**Liquidity state derivation**

```text
bestBid null, bestAsk null → empty
bestBid set, bestAsk null → bid-only
bestBid null, bestAsk set → ask-only
both set → two-sided
```

**UI copy (canonical strings)**

| State | Best bid line | Best ask line | Strip extra |
|-------|---------------|---------------|-------------|
| two-sided | formatted price | formatted price | — |
| bid-only | formatted price | No asks | — |
| ask-only | No bids | formatted price | — |
| empty | No bids | No asks | No market — waiting for liquidity |

**Realtime:** `mapOrderBookUpdatedToSnapshot` already maps `message.bids` / `message.asks` to arrays (possibly empty). `deriveTopOfBook` nulls missing touches — no merge with previous snapshot.

**Logout:** Add to `clearUserScopedQueries`:

```typescript
queryClient.removeQueries({ queryKey: ['market', 'orderbook', 'AAPL'] })
```

**Backend:** No code changes anticipated; integration tests validate contract.
