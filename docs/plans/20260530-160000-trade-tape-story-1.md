## Executive summary

Deliver **Story 1** of US-07: authenticated users see the **50** most recent **AAPL** market executions on the trading view within **2 s**, loaded from `GET /api/market/trades?symbol=AAPL&limit=50`. The read path follows Tech В§10.5 and mirrors US-05: **Redis `trades:AAPL:recent` first** (JSON payload), **PostgreSQL fallback** via `ix_trades_symbol_time` on the existing `trades` table (DB В§4.7). Public rows expose **tradeIdentifier**, **price**, **quantity**, and **executedAt** only вҖ” no buyer/seller IDs (BR-04, BR-08).

The matching engine is still a stub (no `ProcessMatch` or Redis tape **writes** yet), so integration tests and local demos rely on **PostgreSQL seeding** until Story 2 wires match вҶ’ projection вҶ’ SignalR. Frontend adds a **trade tape panel** below the order book depth in the left workspace column, with loading skeleton, retryable error state, and Time / Price / Size columns вҖ” **no fabricated rows** and **no SignalR prepend** (Story 2).

## Goals and non-goals

**Goals**

- G1: `GET /api/market/trades?symbol=AAPL&limit=50` returns up to **50** trades, **newest first**, for authenticated sessions (`401` without session; `400` for non-AAPL or invalid limit).
- G2: Trading view shows a **Trade tape** panel with readable time, price (up to 4 dp), and whole-share quantity; loads via TanStack Query on first paint.
- G3: Prove AC with Api integration tests (ordering, cap 50, auth, invalid symbol) and a manual Aspire checklist.

**Non-goals** (this plan will not do)

- NG1: SignalR `TradeTapeEntryPublished` client handling or вүӨ500 ms live prepend (**Story 2**, #81).
- NG2: Empty-state copy (вҖңNo trades yet вҖ” waiting for executionsвҖқ) and logout cache flash guard (**Story 3**, #82).
- NG3: Full PRD В§8.1 layout polish, tablet stacking refinements, optional row highlight (**Story 4**, #83).
- NG4: Matching engine `ProcessMatch`, Redis tape **writes**, or `PublishTradeTapeEntryAsync` from match path (needed for live tape; Story 2).
- NG5: User trade history tab changes (`GET /api/trades`), last-trade header (FR-2.2), chart (US-08), aggressor side column.

**PostgreSQL fallback:** Query `trades` by `symbol`, `AsNoTracking`, order `executed_at DESC`, take `limit`. Map `external_id` вҶ’ `tradeIdentifier`; omit `buyer_user_id`, `seller_user_id`, order IDs (BR-04).

**Redis preferred read:** Single JSON key (same pattern as `orderbook:{symbol}:snapshot`) containing `{ symbol, trades[], updatedAt }`. Deserialize, validate symbol, trim to requested limit. On miss or parse failure, fall back to PG without failing the request (EC-04).

## Authorization, session, and domain notes

- **Session model:** Cookie session; `ICurrentUserAccessor.UserId` null вҶ’ `401` in handler (same as `GetOrderBookSnapshotQueryHandler`).
- **Route protection:** `/trading` behind `ProtectedRoute`; query `enabled: authStatus === 'authenticated'`.
- **Domain rules (implementers must not violate):**
  - BR-02: Default and max `limit` = **50**.
  - BR-03: `executedAt` descending (newest first).
  - BR-04: Response fields: `tradeIdentifier`, `price`, `quantity`, `executedAt` only.
  - BR-08: Market-wide tape вҖ” no `userId` filter on read path.
  - EC-07: Non-AAPL symbol вҶ’ `400` + `INVALID_SYMBOL`.
  - Never return placeholder prices on error or empty failure body.

## Implementation details (for /build)

**Handler validation (`GetRecentTradesQueryHandler`):**

- `SupportedSymbol = "AAPL"`; normalize trim + uppercase.
- `limit`: default **50**; if `<= 0` or `> 50` вҶ’ `400` `INVALID_LIMIT` (or clamp default only when omitted вҖ” prefer explicit reject for `> 50` per EC-07 spirit).
- Map read model вҶ’ `RecentTradeItemResponse(tradeIdentifier, price, quantity, executedAt)`.

**Error codes:**

- `401` `UNAUTHORIZED` вҖ” no session.
- `400` `INVALID_SYMBOL` вҖ” not AAPL.
- `400` `INVALID_LIMIT` вҖ” outside 1вҖ“50.

**PG query (fallback):**

```text
trades.AsNoTracking()
  .Where(t => t.Symbol == symbol)
  .OrderByDescending(t => t.ExecutedAt)
  .Take(limit)
  .Select(t => new RecentTradeReadModel(t.ExternalId, t.Price, t.Quantity, t.ExecutedAt))
```

**Frontend query:**

```typescript
queryKey: ['market', 'trades', 'AAPL']
queryFn: ({ signal }) => getRecentTrades('AAPL', 50, signal)
enabled: authStatus === 'authenticated'
staleTime: 0
suppressErrorToast: true on fetch (match order book)
```
