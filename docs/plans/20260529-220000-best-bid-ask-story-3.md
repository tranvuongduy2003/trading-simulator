## Executive summary

Deliver **Story 3** of US-05: extend the existing top-of-book strip so logged-in users see **Spread** (absolute USD + percentage) and **Mid** (USD) derived from best bid and best ask per **BR-03** and **BR-04**. All computation stays **client-side** on the same TanStack Query snapshot used by Stories 1–2; no new REST or SignalR contracts. Values recompute on initial `GET /api/market/orderbook` and on each `OrderBookUpdated` cache update. One-sided or empty books show **—** for spread/mid (never `0.00` placeholders).

Stories 1–2 (`deriveTopOfBook`, `TopOfBookStrip`, hub `setQueryData`, reconnect badge) are the foundation; this plan only extends `web/src/features/market/*`.

## Goals and non-goals

**Goals**

- G1: When both touch prices exist, show spread absolute, spread %, and mid per spec AC (example: bid **150.00**, ask **150.10** → spread **$0.10**, **0.067%**, mid **$150.05**).
- G2: Text labels **Spread** and **Mid** with `tabular-nums` styling (PRD §8.2).
- G3: One-sided book (bid-only or ask-only) → spread and mid **—** (EC-02).
- G4: Real-time: spread/mid update with bid/ask on `OrderBookUpdated` without invalidating wallet/portfolio queries.
- G5: Manual Aspire checklist signed off.

**Non-goals** (this plan will not do)

- NG1: Last trade price or daily change in header (**FR-2.2**, separate delivery).
- NG2: “No bids” / “No market” marketing copy (**Story 4**, #61).
- NG3: Backend spread/mid fields on API or hub payload.
- NG4: Full depth table, chart, tape.
- NG5: Frontend unit-test harness (Vitest) — out of MVP scope; derivation kept pure in `top-of-book-display.ts` for future tests.

**Derivation (BR-03, BR-04)** — implement in `top-of-book-display.ts`:

| Field | Formula | When defined |
|-------|---------|--------------|
| `spreadAbsolute` | `bestAsk.price − bestBid.price` | Both touches non-null |
| `midPrice` | `(bestBid.price + bestAsk.price) / 2`, rounded to **4** decimal places | Both touches non-null |
| `spreadPercent` | `(spreadAbsolute / midPrice) × 100`, display rounded to **3** decimal places | Both touches non-null and `midPrice > 0` |

Example validation: bid `150.00`, ask `150.10` → spread `0.10`, mid `150.05`, percent `≈ 0.067%`.

Use `formatPrice` for USD absolute and mid; add `formatSpreadPercent(value)` (e.g. `0.067%`) without currency symbol.

**Float safety:** Round internal mid and spread absolute to 4 dp before percent calculation to align with NUMERIC(18,4) intent:

```text
round4(x) = Math.round(x * 10000) / 10000
```

## Implementation details (for /build)

**Types (`top-of-book-display.ts`):**

```text
TopOfBookSpread = { absolute: number; percent: number }
TopOfBookDisplay += spread: TopOfBookSpread | null; midPrice: number | null
```

**`deriveTopOfBook`:** After resolving `bestBid` / `bestAsk`, if either is null → `spread: null`, `midPrice: null`. Else compute with `round4`.

**Formatters:**

- `formatSpreadAbsolute(spread)` → `formatPrice(spread.absolute)` or **—**
- `formatSpreadPercent(spread)` → `` `${percent.toFixed(3)}%` `` or omit second line
- `formatMidPrice(mid)` → `formatPrice(mid)` or **—**

**Strip:** Reuse `missingValueLabel` (**—**) for undefined spread/mid; do not show `N/A` string unless product prefers later (open question resolved: **—**).

**Hub path:** No code change required if `deriveTopOfBook` is invoked from `trading-page` on `orderBookQuery.data` — verified in Story 2.
