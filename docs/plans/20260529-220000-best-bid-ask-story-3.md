## Executive summary

Deliver **Story 3** of US-05: beside best bid and best ask on the trading view, show **Spread** (absolute USD and percentage) and **Mid** (USD average of touch prices) derived **client-side** from the same snapshot/real-time book levels as Stories 1–2 (BR-03, BR-04). No new API or SignalR contract. When either touch is missing (one-sided or empty book), spread and mid show **—** (never `0.00` or a fabricated mid). Numeric values use `tabular-nums` and explicit text labels per PRD §8.2 / FR-2.5.

**Prerequisite:** Story 1 merged (`GET /api/market/orderbook`, `TopOfBookStrip`, `deriveTopOfBook`). Story 2 (#59) is not required for Story 3 AC on snapshot load; when Story 2 lands, metrics must recompute from the same `TopOfBookDisplay` source without extra wiring.

## Goals and non-goals

**Goals**

- G1: Implement BR-03/BR-04 in pure frontend helpers with 4-decimal internal rounding for money values.
- G2: Extend `TopOfBookStrip` to show labeled **Spread** and **Mid** beside bid/ask with tabular styling.
- G3: Prove AC via manual checklist (worked example bid 150.00 / ask 150.10 → spread $0.10, ~0.067%, mid $150.05) and one-sided N/A cases.

**Non-goals** (this plan will not do)

- NG1: New REST endpoint or server-side spread/mid fields.
- NG2: Empty-book marketing copy (“No bids”) — **Story 4** (#61).
- NG3: SignalR hub or `OrderBookUpdated` pipeline changes (**Story 2**, #59) — only document that metrics follow the same display input when Story 2 merges.
- NG4: Last trade price (FR-2.2).
- NG5: Automated Vitest/frontend test suite (MVP policy); optional follow-up if team opts in.

  - **EC-02:** One-sided book → spread and mid **—** (not zero).

#### Objective

Pure functions compute spread absolute, spread percent, and mid from touch prices; `deriveTopOfBook` (or adjacent export) returns metrics alongside bid/ask. One-sided input yields `null` metrics.

#### Implementation notes

- Add `TopOfBookMetrics` (or embed on `TopOfBookDisplay`):

  ```text
  spreadAbsolute: number | null
  spreadPercent: number | null   // raw ratio × 100, not formatted
  midPrice: number | null
  ```

- `computeTopOfBookMetrics(bestBid, bestAsk)`:
  - If either touch is `null` → all metrics `null`.
  - `midPrice = roundTo4((bid + ask) / 2)`
  - `spreadAbsolute = roundTo4(ask − bid)`
  - `spreadPercent = mid > 0 ? roundPercent3((spreadAbsolute / mid) * 100) : null`
- **Worked example:** bid `150`, ask `150.1` → mid `150.05`, spread `0.1`, percent `0.067` (display `0.067%`).
- Add `formatSpreadAbsolute`, `formatSpreadPercent`, `formatMidPrice` — reuse `formatPrice` for USD fields; percent as `` `${value.toFixed(3)}%` `` (no `$`).
- Extend `deriveTopOfBook` to attach metrics so callers (page or strip) have a single object.

## Implementation details (for /build)

**Computation (reference):**

```text
roundTo4(x) = Math.round(x * 10000) / 10000
mid = roundTo4((bid + ask) / 2)
spreadAbsolute = roundTo4(ask - bid)
spreadPercent = mid > 0 ? roundTo3((spreadAbsolute / mid) * 100) : null
```

**Display:**

| Field | Formatter | Example |
|-------|-----------|---------|
| Spread absolute | `formatPrice(spreadAbsolute)` | `$0.10` |
| Spread % | `` `${spreadPercent.toFixed(3)}%` `` | `0.067%` |
| Mid | `formatPrice(midPrice)` | `$150.05` |
| Missing | literal `—` | `—` |

**Integration point:** `TradingPage` already sets `topOfBookDisplay = deriveTopOfBook(orderBookQuery.data)`. After Task 1, `TopOfBookDisplay` includes metrics; strip reads `display.spread` / `display.mid` (or nested `metrics`).

**Story 2 note:** When `OrderBookUpdated` updates query cache, `deriveTopOfBook` re-runs → metrics refresh automatically. Do not add spread/mid fields to SignalR payload.
